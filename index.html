Introduction

This page was created by Yamkela Lerotholi to demonstrate the use of semantic HTML5, accessibility techniques, and SEO-friendly design principles. It is fully structured using HTML5 elements without any CSS or JavaScript.

Why Semantic HTML Matters

1. Improved Accessibility

Semantic tags like <header>, <nav>, and <section> help assistive technologies understand the layout of the page, making content more accessible for everyone.

2. Better SEO

Search engines read semantic structure to determine what your content is about. By organizing your page with proper headings and descriptions, it becomes easier to rank and be found online.

3. Cleaner Code

Using semantic elements makes your code easier to read, maintain, and scale. It also helps other developers (or future-you) quickly understand the structure.
